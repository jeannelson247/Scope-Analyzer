1. Change Set 01 - Added formula execution layer
File: /Users/jeannelson/Desktop/scope_studio2/signal_tools.py
Why this file exists:
- The original app could only do linear gain/offset conversion.
- Your MATLAB workflow needs busbar formulas, baseline subtraction, filtering, Rogowski/B-dot integration, and future custom expressions.

Line-by-line explanation
- [17] `class FormulaError(ValueError):` Added a dedicated exception type so formula failures can be caught cleanly in the GUI and shown to the user without crashing the app.
- [21-31] `baseline(...)` Added a reusable baseline subtraction helper because your MATLAB scripts repeatedly subtract the mean of a pre-shot time window.
- [23-24] `np.asarray(..., dtype=np.float64)` Forces predictable numeric typing so formulas behave consistently across large CSV files.
- [25-28] `mask = ...` Supports both "everything before time X" and "between start and end" baseline windows.
- [29-31] `if not np.any(mask) ... return y - mean(...)` Keeps the function safe when no baseline samples exist and only subtracts when a real baseline window is available.
- [34-36] `demean(...)` Added a simple whole-trace mean removal helper because it is a common signal cleanup primitive.
- [39-45] `movmean(...)` Added moving-average smoothing so MATLAB-style smoothed traces can be expressed directly in the formula field.
- [41] `window = max(int(window), 1)` Prevents invalid zero or negative window sizes.
- [44-45] `kernel ... np.convolve(...)` Implements the moving-average itself with a small dependency-free method.
- [48-53] `gradient(...)` Added derivative support for quick waveform-rate calculations and future diagnostic formulas.
- [51-53] small-array guard + `np.gradient(...)` Prevents errors on short traces and uses NumPy's tested derivative logic.
- [56-66] `integrate(...)` Added cumulative trapezoidal integration because Rogowski and B-dot analysis often requires integrating voltage-like signals over time.
- [62-65] `seg ... np.cumsum(seg)` Implements trapezoidal integration in a SciPy-independent way.
- [66] `return -out if negate else out` Mirrors the sign flip used in your MATLAB integrated-Bdot workflow.
- [69-79] `_butter_lowpass(...)` Added a high-quality low-pass path using SciPy when available so formulas can reproduce MATLAB-like filtered signals more closely.
- [72-74] `try import scipy.signal ... except` Keeps the feature optional rather than making the whole app depend on SciPy.
- [75-79] Nyquist normalization plus `butter` / `filtfilt` gives a stable zero-phase filter when SciPy exists.
- [82-95] `_rc_lowpass(...)` Added a fallback filter so low-pass formulas still work even if SciPy is not installed.
- [83-86] cutoff and `dt` validation prevents divide-by-zero or nonsense filters.
- [87-95] forward/backward RC smoothing provides a lightweight approximate low-pass instead of failing outright.
- [98-111] `lowpass(...)` Created the public filter helper that all formulas call.
- [99-107] Converts inputs and computes a robust sample spacing from valid time deltas.
- [108-111] Uses SciPy filtering if possible, otherwise automatically falls back to the RC approximation.
- [114-134] `SAFE_FUNCS = {...}` Explicitly whitelisted only safe numerical helpers so user-entered formulas can do useful signal math without executing arbitrary Python.
