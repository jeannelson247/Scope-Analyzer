Current Assessment
------------------
Scope Studio is already a strong desktop analysis tool for large oscilloscope
CSV files. Its best design choices are:
- The CSV loader is tolerant of Tektronix-style preambles and peak-detect
  companion columns.
- Plotting stays responsive because pyqtgraph handles display downsampling
  while exports can still use higher-resolution data.
- Conversion formulas are sandboxed, so users can type MATLAB-like formulas
  without giving the formula field arbitrary Python execution.
- Calibration is separated into two layers: fixed conversion presets for
  known sensors, and data-dependent reference fitting for shot-specific
  calibration ratios.
- The side chat is correctly designed as an assistant and task router, while
  deterministic NumPy/SciPy code performs the numerical work.

Main improvement opportunities identified:
- Add a direct low-pass filter control to the GUI instead of requiring a
  lowpass(...) expression in every channel formula.
- Add a larger formula editor with validation and preset saving, because
  long calibration formulas are awkward in a table cell.
- Encode the MATLAB calibration formulas explicitly as reusable presets.
- Make moving-average formulas sample-rate aware.
- Keep formula and anomaly tools safe when no file is loaded.
- Document the project history in one place.

- presets.json:98-103

- presets.json:110-115
  Name: MATLAB busbar 750 A/V from (2.5 - V)
  Formula: (2.5 - x) * -750
  Why: Preserves the MATLAB polarity form used in several busbar scripts.

- presets.json:116-121
  Name: MATLAB small-channel 100 A/V from (2.5 - V)
  Formula: (2.5 - x) * -100
  Why: Preserves the small-channel current monitor conversion from the MATLAB
  code.

- presets.json:122-127
  Name: MATLAB large loop channel 1500/4 A/V
  Formula: x * (1500 / 4)
  Why: Mirrors I_CM1, I_CM2, and I_loop scaling in the later scope scripts.

- presets.json:128-133
  Name: MATLAB Vdiff current, precomputed Vdiff / 0.08 ohm
  Formula: x / 0.08
  Why: Mirrors I_diff = V_difference ./ (8*10^(-2)). Because the current
  formula engine is single-channel, this preset expects x to be a precomputed
  voltage-difference column.

- presets.json:134-139
  Name: MATLAB moving average 5 ms
  Formula: movmean(x, samples(5e-3))
  Why: Mirrors MATLAB movmean windows expressed as time rather than requiring
  the user to manually calculate samples.

- presets.json:140-145
  Name: MATLAB moving average 10 ms
  Formula: movmean(x, samples(10e-3))
  Why: Mirrors 10 ms smoothing windows used in the later TFC scripts.

Important calibration note:
- The MATLAB ratios ratio_CH1 and ratio_CH3/ratio_CH4 are not fixed presets.
  They are shot-dependent values computed from:
  ratio = dot(source_window, reference_window) / dot(source_window, source_window)
- Scope Studio implements this in the Reference calibration panel so the
  result depends on the loaded shot, selected source/reference channels, and
  fit window.

Change Set 60 - FIX blank-plot regression: restore "set view-dependent
                curve options AFTER addItem" ordering (broke in CS53)
Primary files: plot_render.py, app.py, tests/test_engine.py
Implemented by: Claude in this pass.

Symptom (user, with CSVs loaded, channels ticked): the 2D plot renders
nothing - blank white plot, often with a runaway autorange (axes showing
~1e9 / 6e8 and an empty legend box top-left), for both a 125k-row and a
1.25M-row shot.

Root cause (regression introduced by fable's CS53, the plot_render
split): refresh_plot used to do, per a documented invariant -
    curve = pg.PlotDataItem(x, y, ..., skipFiniteCheck=True)
    pi.addItem(curve)                 # add to a ViewBox FIRST
    curve.setDownsampling(auto=True, method="peak")   # THEN tune
    curve.setClipToView(True)
with the comment "set after adding (pyqtgraph needs a parent view
first)". CS53's make_curve folded setDownsampling/setClipToView INSIDE
make_curve, so they ran on a still-detached item (before addItem).
pyqtgraph resolves clip-to-view and auto peak-downsampling against the
item's parent view; with no view yet, clip-to-view can clip the entire
trace to nothing and autorange runs away -> blank plot. (CS56 only moved
the pyqtgraph import inside make_curve; it did not introduce or fix this.)

Diff proof: the ONLY change to refresh_plot between the last known-good
version (backups/app_before_ui_restore_20260613_045513.py) and now was
the switch to make_curve plus the loss of the two post-addItem lines.

Fix:
1. plot_render.make_curve: now ONLY builds the styled PlotDataItem
   (pen/name/skipFiniteCheck + optional pre-decimation). It no longer
   calls setDownsampling/setClipToView. Docstring warns these are
   view-dependent.
2. plot_render.enable_view_downsampling(curve): new helper applying
   setDownsampling(auto=True, method="peak") + setClipToView(True),
   to be called AFTER addItem.
3. app.py refresh_plot: imports enable_view_downsampling and calls it on
   each curve right after vb_right.addItem / pi.addItem. This reproduces
   the exact known-good ordering. make_curve has a single caller, so no
   other path is affected.

Why this is safe: behavior now matches the pre-CS53 working version
byte-for-byte in ordering and arguments (skipFiniteCheck=True, no
pre-decimation in the 2D path, auto peak-downsampling for full-res zoom).

Verification:
- python3 -m py_compile app.py plot_render.py pass.
- plot_render imports headless; make_curve / enable_view_downsampling /
  decimate_for_render all present.
- New headless regression test (test_enable_view_downsampling_contract)
  asserts the helper applies exactly {auto:True, method:peak} + clip=True
  via a recording stub, so re-folding them into make_curve (pre-addItem)
  would now fail CI. pytest -> 17 passed.
- NOTE: the actual on-screen render still needs a visual check on the
  user's Mac (Qt cannot run in this headless sandbox); the fix restores
